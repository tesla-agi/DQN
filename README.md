# DQN — Deep Q-Network (Dueling DQN + Double DQN)

A clean, from-scratch implementation of DQN and its two headline improvements
(Double DQN, Dueling DQN), built for **LunarLander-v3**. Part of a structured deep RL
curriculum — the goal is textbook-depth understanding paired with production-quality code.

## Why DQN needs its two stabilizers

Naive Q-learning with a neural network diverges. Two problems, two fixes:

- **Correlated samples.** Consecutive transitions within an episode are highly
  correlated, which breaks the i.i.d. assumption SGD relies on. **Experience replay**
  stores transitions in a buffer and samples random minibatches, breaking the correlation.
- **Moving targets.** The TD target `r + γ·max Q(s')` is computed by the same network
  being updated, so every gradient step also moves the target you are chasing — training
  chases its own tail. A **target network** (a frozen copy synced every C steps) holds the
  target still between syncs.

These are separate mechanisms: the buffer fixes *what data you learn from*, the target
network fixes *what you compare against*.

## Improvements (both toggleable)

- **Double DQN** (`use_double_dqn=True`): vanilla DQN's single `max` both selects and
  evaluates the next action with one network, so noise-inflated actions get systematically
  favored → overestimation bias. Double DQN decouples the two jobs — the online net
  *selects* (`argmax`), the target net *evaluates* (`gather`) — so the bias largely cancels.
- **Dueling DQN** (`dueling=True`): splits the Q-head into a state-value stream `V(s)` and
  an advantage stream `A(s,a)`, recombined as `Q = V + (A − mean(A))`. The mean-subtraction
  is the identifiability fix (without it, V and A are not uniquely determined).

## Key design choices

- **State-only network.** DQN takes only the state and outputs a Q-value for *every*
  action at once; the action is just an index into that output. This makes the `argmax`
  cheap — the whole reason discrete-action DQN works without a separate policy network.
- **`done` stored as float32**, not bool — it is used arithmetically in the TD target
  (`target = r + γ·Q(s')·(1 − done)`) to zero out bootstrapping on terminal transitions.
- **Hard target sync** every C steps (`sync_target`), not soft/Polyak updates.
- **Gradient clipping** by norm, to guard against occasional large TD errors.

## Ablations

Each is a single flag flip on the same agent:

| Ablation      | Flag                   | Expected effect                            |
|---------------|------------------------|--------------------------------------------|
| No replay     | `correlated=True`      | Unstable / degraded training               |
| No Double DQN | `use_double_dqn=False` | Q-value overestimation (visible in mean-Q) |
| No dueling    | `dueling=False`        | Baseline (often little change on this task) |
